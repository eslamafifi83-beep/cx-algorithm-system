4. Changing DESIGN (UX/UI) — the code route

Design lives in the code. The workflow is always the same three steps:

  1. Edit the files on your computer: C:\Users\eslam\MOCKUPS\cx-algorithm-system — easiest way: open Claude Code in that folder and describe the change ("make the hero headline bigger", "change the gold to a warmer tone"). Claude has memory notes about this project's whole history.

5. Things that will bite if forgotten ⚠️

  • Git author identity: code pushed to GitHub must be authored as 298448529+eslamafifi83-beep@users.noreply.github.com or Vercel blocks the deployment ("commit author does not have access"). This is already configured inside the project folder — but if the project is ever copied to a new computer, set it again first thing.
  • Don't delete vercel.json: it pins the project type ("Other") — without it Vercel hunts for a server file and the build fails with "No entrypoint found."
  • Never commit .env: it's git-ignored on purpose. The one on your machine currently holds a dead (rotated) key — harmless.
  • Cloudflare's orange cloud: DNS records pointing at Vercel must stay gray ("DNS only"). Turning them orange breaks the site.
  • Domain renewal: thecxalgorithm.com renews yearly (~$11) — keep Auto-Renew ON and your card valid in Cloudflare, or the site (and email) go dark.
  • Vercel free plan is for non-commercial use — fine for the podcast; if you start selling sponsorships through the site, upgrade to Pro (~$20/mo).

6. If something breaks — first aid

SymptomDo this
Site is down / broken after a changeVercel → Deployments → find the last working deployment → ⋯ menu → "Promote to Production" (or "Instant Rollback"). Instantly restores the previous version.
Forgot admin passwordVercel → Settings → Environment Variables → edit ADMIN_PASSWORD → save → Deployments → Redeploy.
Edits in admin don't appear on the siteWait ~30 seconds and hard-refresh (Ctrl+Shift+R) — pages briefly cache content.
Deployment "Blocked — commit author…"The git identity issue above. Re-set the author email in the project folder.
Anything elseOpen Claude Code in the project folder and describe/screenshot the problem — the project history is in its memory.
