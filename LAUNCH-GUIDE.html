Launch Guide — putting your website online

A complete, beginner-friendly, step-by-step plan using Vercel + Supabase. You've never done this before, and that's completely fine.

👤 YOU = something you click in a website. I'll help at every step. 🤖 CLAUDE = code & setup I do for you. You don't touch these.

You never have to write code. Your job: make a few free accounts, paste in some values I give you, and click "Deploy." My job: everything technical. Your time: roughly 1–1.5 hours across a few sittings. No rush, nothing risky until the final step.

The big picture, in plain words

Right now your website lives only on your computer. To put it online we use two free services:

  • Vercel = the stage. It shows your pages to the world and runs the behind-the-scenes logic (admin panel, contact form).
  • Supabase = the filing cabinet. It permanently stores your content, uploaded images/audio, and the messages people send you.

You edit in your admin panel → it saves to Supabase → your live pages read from Supabase. Two more helpers come later: GitHub (a safe online copy of the code that Vercel reads from — so your site auto-updates) and Cloudflare (points your web address at the site and gives free email forwarding).

What it costs

~$12 / year  to start — just the domain name.

  • Vercel: Free. (Free plan is for non-commercial sites; a personal podcast is fine. Direct sponsorship sales later → ~$20/mo Pro.)
  • Supabase: Free — includes 1 GB storage (your images + ~15–40 audio episodes). Video is huge, so we use Spotify embeds for video instead of storing it.
  • GitHub & Cloudflare: Free.
  • Domain: ~$12/year.   Email: free forwarding, or ~$6/mo for a full inbox (optional).

Phase 1 — Make your accounts (👤 you, ~15 min)

Make three free accounts. Use the same email for all, and choose "Continue with GitHub" where offered — it links everything and saves time.

1.1 — GitHub (do this first)

Go to github.com → Sign up → enter email, password, a username (e.g. eslam-afifi) → verify your email. You don't need to understand GitHub; it's just a safe home for the code.

1.2 — Supabase

Go to supabase.com → Start your project → Continue with GitHub → approve. Don't create a project yet — we do that together in Phase 2.

1.3 — Vercel

Go to vercel.com → Sign Up → choose Hobby (free) → Continue with GitHub → approve. Don't import anything yet.

✅ When all three say you're logged in, Phase 1 is done — tell me.

Phase 2 — Set up Supabase (👤 clicks + 🤖 me)

2.1 — 👤 Create the project

  1. Click New project.
  2. Name: cx-algorithm
  3. Database Password: click Generate a password, then copy & save it somewhere safe.
  4. Region: pick the one closest to most of your listeners (e.g. West EU / Central EU). Unsure? Ask me.
  5. Click Create new project and wait ~2 minutes.

2.2 — 👤 Send me your project's public keys

Click the gear icon (Project Settings) → API. Copy me the first two only: the Project URL and the anon public key.

🔒 Do NOT send me the service_role key. It's the master key. You'll paste it directly into Vercel yourself in Phase 4.

2.3 — 🤖 I build your filing cabinet

I'll give you a block of text to paste into Supabase's SQL Editor and click Run (one click). That creates the tables and storage folders, and I'll load your current website content in so nothing is lost.

Phase 3 — I rebuild the code (🤖 me — you relax)

Nothing for you to do. I convert your behind-the-scenes server to Vercel's format, point your pages at Supabase, wire the admin panel to save to the cloud, set up image + large audio/video uploads, and replace the admin password with a secure secret you'll set yourself. This is the part that takes me a few work sessions — I'll keep you posted.

Phase 4 — Deploy to Vercel (👤 you click, I guide)

4.1 — 👤 Import the project

In Vercel: Add New… → Project → find your repo (cx-algorithm-system) → Import.

4.2 — 👤 Add the secret settings

In Environment Variables, add these (I'll give you exact values to paste):

  • SUPABASE_URL — your project URL
  • SUPABASE_ANON_KEY — the public key
  • SUPABASE_SERVICE_ROLE_KEY — 🔒 the master key you kept private (paste here, only here)
  • ADMIN_USER — a username you choose
  • ADMIN_PASSWORD — a strong new password (replaces cxalgorithm2026)

4.3 — 👤 Deploy

Click Deploy, wait ~1 minute → you get a live link like cx-algorithm-system.vercel.app. Your site is online. 🎉 We then test it together — pages, admin login, a test upload, a test message.

Phase 5 — Domain + email (👤 you + 🤖 me)

5.1 — 👤 Register the domain (~$12/yr)

Go to cloudflare.com → free account → Domain Registration → Register Domain → search & buy (sold at cost, no markup).

5.2 — 🤖 + 👤 Connect it

In Vercel: Settings → Domains → type your domain → it shows DNS records. I translate them; you paste them into Cloudflare's DNS page. Shortly after, your domain is live with automatic HTTPS (the padlock).

5.3 — 👤 Email (optional, free)

In Cloudflare: Email → Email Routing → forward hello@yourdomain to your Gmail. Free. (Full send-from inbox = Google Workspace ~$6/mo, later if you want.)

Launch checklist (before you tell the world)

  • Admin password changed to something strong (not the old cxalgorithm2026).
  • You can log into /admin, edit content, and the live site updates.
  • Contact form sends, and the message shows in your admin Messages tab.
  • An uploaded image appears on the live site.
  • The padlock (HTTPS) shows on your domain.
  • Real content added: episode audio/links, guest logos, platform links, newsletter link, gallery photos.
  • Tested on your phone as well as your computer.
